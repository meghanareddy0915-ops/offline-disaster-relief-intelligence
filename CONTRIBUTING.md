# Contributing Guidelines

Thank you for your interest in contributing to the Offline Disaster Relief Intelligence System.

## Getting Started

1. Fork or clone the repository.
2. Create a new branch for your feature or bug fix.
3. Make your changes.
4. Test your changes locally.
5. Commit your changes using meaningful commit messages.
6. Push your branch and create a merge request.

## Coding Standards

* Follow clean coding practices.
* Write meaningful variable and function names.
* Keep code modular and reusable.
* Add comments where necessary.

## Commit Message Format

Use semantic commit messages:

* feat: Add a new feature
* fix: Fix a bug
* docs: Update documentation
* chore: Maintenance tasks
* refactor: Improve existing code
* test: Add or update tests

Example:

feat: implement OCR text extraction

## Pull Requests

* Keep pull requests focused on a single task.
* Ensure the project builds successfully.
* Update documentation if required.

## Code Review

All contributions should be reviewed before merging into the main branch.

Thank you for contributing!
